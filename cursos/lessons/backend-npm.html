NPM e Gerenciamento de Pacotes

Módulo 1 Aula 2 30 minutos

O que é NPM?

NPM (Node Package Manager) é o gerenciador de pacotes padrão do Node.js. Permite instalar, compartilhar e gerenciar bibliotecas JavaScript de forma eficiente.

Inicializando um Projeto

$ npm init
# ou para usar configurações padrão
$ npm init -y

Este comando cria o arquivo package.json, que contém as informações e dependências do projeto.

Comandos Essenciais do NPM

Instalar Pacotes

$ npm install express
$ npm i lodash --save
$ npm i nodemon --save-dev

--save: Dependência de produção
--save-dev: Dependência de desenvolvimento

Remover Pacotes

$ npm uninstall express
$ npm remove lodash
$ npm rm nodemon --save-dev

Remove o pacote e atualiza o package.json automaticamente.

Atualizar Pacotes

$ npm update
$ npm update express
$ npm outdated

Atualiza pacotes para versões mais recentes compatíveis.

Listar Pacotes

$ npm list
$ npm ls --depth=0
$ npm list -g

Lista pacotes instalados no projeto ou globalmente.

Estrutura do package.json

{
  "name": "meu-projeto",
  "version": "1.0.0",
  "description": "Meu primeiro projeto Node.js",
  "main": "index.js",
  "scripts": {
    "start": "node index.js",
    "dev": "nodemon index.js",
    "test": "jest"
  },
  "dependencies": {
    "express": "^4.18.0"
  },
  "devDependencies": {
    "nodemon": "^2.0.0"
  }
}

Scripts NPM

Executando Scripts

$ npm start
$ npm run dev
$ npm test
$ npm run build

Scripts personalizados definidos no package.json para automatizar tarefas.

Versionamento Semântico

^1.2.3

Aceita mudanças compatíveis
(1.2.3 até 1.x.x)

~1.2.3

Aceita apenas patches
(1.2.3 até 1.2.x)

1.2.3

Versão exata
(apenas 1.2.3)

Pacotes Populares

Express.js

Framework web minimalista e flexível

$ npm install express

Lodash

Biblioteca de utilitários JavaScript

$ npm install lodash

Axios

Cliente HTTP baseado em promises

$ npm install axios

Nodemon

Reinicia automaticamente o servidor

$ npm install -g nodemon

Boas Práticas

  • Sempre use package-lock.json para versões exatas
  • Mantenha dependências atualizadas regularmente
  • Use --save-dev para ferramentas de desenvolvimento
  • Evite instalar pacotes globalmente desnecessariamente
  • Use npm audit para verificar vulnerabilidades
